# HanPocket 위챗 미니프로그램 개발 가이드

## 📋 개요

이 프로젝트는 기존 HanPocket 웹앱을 위챗 미니프로그램으로 래핑하여 중국 시장에서 서비스를 제공하기 위한 종합 가이드입니다.

## 📁 프로젝트 구조

```
wechat-miniprogram/
├── docs/                           # 문서들
│   ├── 01-developer-account-registration.md    # 개발자 계정 등록 가이드
│   ├── 02-webview-wrapper-design.md           # WebView 래핑 설계
│   ├── 03-deployment-process.md               # 배포 프로세스
│   ├── 04-templates-and-configs.md           # 파일 템플릿
│   └── 05-project-setup.md                  # 개발 환경 설정
├── templates/                      # 설정 템플릿
└── hanpocket-miniprogram/         # 실제 미니프로그램 프로젝트
    ├── app.js                     # 앱 메인 로직
    ├── app.json                   # 앱 설정
    ├── pages/                     # 페이지들
    ├── utils/                     # 유틸리티
    └── components/                # 컴포넌트들
```

## 🚀 시작하기

### 1. 개발자 계정 준비
- [개발자 계정 등록 가이드](docs/01-developer-account-registration.md) 참고
- 중국 신분증 (범범뻠 와이프) 필요
- 위챗 공중호 플랫폼에서 계정 생성

### 2. 개발 환경 설정
- [개발 환경 설정 가이드](docs/05-project-setup.md) 참고
- 위챗 개발자 도구 설치
- Node.js 환경 구성 (선택사항)

### 3. 프로젝트 개발
- [WebView 래핑 설계](docs/02-webview-wrapper-design.md) 참고
- HanPocket 웹앱과 미니프로그램 간 브리지 구현
- 위챗 네이티브 기능 통합 (로그인, 결제, 공유)

### 4. 배포
- [배포 프로세스](docs/03-deployment-process.md) 참고
- 심사 제출 및 승인 대기
- 정식 릴리즈

## 🔧 핵심 기능

### 위챗 네이티브 기능

- 미니프로그램 전용 UI/UX

### 성능 최적화
- 지연 로딩 및 코드 분할
- 이미지 최적화
- 캐싱 전략
- 오프라인 지원

- 토큰 만료 및 갱신 처리

### 규정 준수
- 중국 인터넷 규정 엄격 준수
- 개인정보 보호법 준수
