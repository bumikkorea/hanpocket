## 2. 상단바 고정 해제 (App.jsx)
현재 상단바: `<div className="relative z-10" ...>` — relative라 고정 안 됨.
하지만 콘텐츠 영역과 상단바 사이에 뭔가 문제가 있을 수 있음.
상단바가 sticky나 fixed가 되지 않도록 확인. `position: sticky`나 `position: fixed`가 절대 적용되지 않아야 함.
만약 부모 요소 중에 `overflow-hidden` 같은 게 있어서 스크롤이 이상하게 되는 거면 그것도 수정.

핵심: 콘텐츠 전체(상단바 포함)가 하나의 스크롤 영역이어야 함. 상단바를 터치해서 아래로 스와이프하면 상단바도 같이 올라가야 함.

## 3. 인사말 폰트 크기 축소 (HomeTab.jsx)
현재: `<p className="px-4 text-2xl font-light mb-4" ...>`
변경: `<p className="px-4 text-base font-light mb-2" ...>`
(text-2xl → text-base, mb-4 → mb-2)

## 4. 상단 여백 축소 (App.jsx)
현재 콘텐츠 영역: `<div className="pt-4 pb-4">`
변경: `<div className="pt-1 pb-4">`
(pt-4 → pt-1)

## 5. 탭 전환 시 스크롤 위치 5분간 기억 (App.jsx)

### 구현 방법:
1. AppInner에 `useRef`로 스크롤 위치 맵 저장:
```js
const scrollPositions = useRef({})  // { tabId: { y: number, timestamp: number } }
```

2. 콘텐츠 영역에 ref 추가:
```js
const contentRef = useRef(null)
```
하지만 실제로는 window 스크롤을 사용할 수 있음. 메인 스크롤러가 window인지 특정 div인지 확인 필요.

3. 탭 전환 시:
```js
// 기존 탭 스크롤 위치 저장
const handleTabChange = (newTab) => {
  scrollPositions.current[tab] = { y: window.scrollY, timestamp: Date.now() }
  setTab(newTab)
  setSubPage(null)
  
  // 새 탭 스크롤 위치 복원 (5분 이내면)
  const saved = scrollPositions.current[newTab]
  if (saved && (Date.now() - saved.timestamp) < 300000) { // 5분 = 300000ms
    requestAnimationFrame(() => {
      requestAnimationFrame(() => {
        window.scrollTo(0, saved.y)
      })
    })
  } else {
    window.scrollTo(0, 0)
  }
}
```

4. **bottomTabs 클릭 핸들러를 handleTabChange로 교체**:
현재 bottomTabs onClick: `() => { setTab(item.id); setSubPage(null) }`
변경: `() => handleTabChange(item.id)`

5. **HomeTab 등 내부에서 setTab 호출하는 경우**: 그건 서브페이지 이동이므로 스크롤 저장 불필요. bottomTabs 클릭만 처리하면 됨.

주의: 만약 메인 스크롤러가 window가 아니라 특정 div라면, 해당 div의 scrollTop을 사용해야 함. `min-h-screen` div가 메인이라면 window.scrollY 사용.
