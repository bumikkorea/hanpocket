# 온보딩 회색지대: "도착 후 첫 1시간" 문제

## 발견 배경
**2026-03-24** — 사용자가 한국 공항 도착 직후 경험

## 문제의 핵심

### 1️⃣ 중국 구매 아이폰 → eSIM 불가능
- 중국 제조 아이폰은 eSIM 기능이 **제조사 차원에서 비활성화**되어 있음
- 이유: 중국 통신사/정부 정책
- 결과: eSIM 가입 불가능

### 2️⃣ 한국 여행용 USIM → 본인인증 불가능 (CRITICAL)
- 공항 T1/T2에서 구매 가능한 "여행용 USIM"은 한국 휴대폰번호 할당받음
- **하지만:** 본인인증(SMS, 카카오톡 인증, 은행 인증 등) **불가능**
- 원인: 본인인증이 필요한 USIM은 **등록된 외국인(D-2, F-2, etc.)만 가능**
  - 관광객(무비자 또는 관광비자)은 "등록되지 않은 외국인" = 절대 불가능
  - 이는 **한국 통신사 정책상 완전한 회색지대**

### 3️⃣ 결과: 앱 진입 불가
```
아이폰 도착 → eSIM 안됨 → 공항 여행용 USIM 구매
  → 휴대폰번호 할당 → 하지만 본인인증 실패
  → 모든 한국 앱 진입 불가 (카카오톡, 뱅킹, HanPocket 등)
  → 택시/지도는? → 카카오맵 (회원가입 필수 아님, 비회원 OK)
  → 경쟁 앱으로 분산 ❌
```

## 비즈니스 임팩트

| 타이밍 | 사용자 상태 | HanPocket 역할 | 현재 | 필요 |
|--------|-----------|----------------|------|------|
| **도착 직후** | 휴대폰 인증 불가능 | 모든 기능 접근 불가 | ❌ 막힘 | ✅ 비회원 진입 |
| **1시간 후** | 선택 완료 | 지도/SOS/정보 제공 | ❌ 다른 앱 사용중 | ⏰ 너무 늦음 |
| **호텔 체크인 후** | 한국 친구 빌려서 인증 | 로그인 가능 | ✅ OK | ✅ OK |

**결론:** 첫 1시간이 **생사를 가르는 시간**. 이 시간에 앱이 없으면 Kakao/Naver로 분산.

## 해결책

### 1. OnboardingFlow.jsx — "로그인 없이 시작" 모드
```jsx
// OnboardingFlow.jsx에 추가
<button onClick={() => startBrowsingWithoutLogin()}>
  로그인 없이 둘러보기
</button>
```

### 2. 비회원 접근 가능한 기능 (MVP 최소)
- ✅ 지도 보기 (카카오맵)
- ✅ SOS 탭 (112/119/1345 원클릭)
- ✅ 병원 찾기
- ✅ 관광지 정보 (읽기만)
- ✅ 긴급 연락처
- ❌ 커뮤니티 (로그인 필수)
- ❌ 여행계획 저장 (로그인 필수)

### 3. 나중 인증 패턴
```jsx
// 사용자 여정
도착 → "로그인 없이 시작" → 지도/SOS 사용
  → 호텔 체크인 후 → "지금 계정 만들기" 프롬프트
  → 이메일/위챗/알리페이로 가입
  → 이후 휴대폰번호 추가 (선택사항)
```

### 4. App.jsx 구조 변경
```javascript
// 현재 (문제)
const isLoggedIn = !!profile; // 로그인 필수
if (!isLoggedIn) return <OnboardingFlow />; // 진입 불가

// 개선
const isGuestMode = !profile && guestModeEnabled;
if (!isLoggedIn && !isGuestMode) return <OnboardingFlow />;
// 비회원 모드 → 일부 탭만 표시 (지도, SOS, 정보)
```

## 장기 전략

### Phase 1 (MVP 필수)
- 비회원 모드: 지도 + SOS + 정보 열람
- 비회원 → 회원: 부드러운 전환 (재참여 카드)

### Phase 2 (P1)
- 비회원도 위시리스트 저장 (로컬스토리지)
- 비회원 여행계획 (로컬)
- 로그인 시 동기화

### Phase 3 (P2+)
- 소셜 로그인 (카카오 필수 X)
- SMS 없는 인증 (위챗, 알리페이, 이메일)
- 오프라인 데이터 프리로드
