# Board-015: 한식가이드 포켓 (KoreanFoodPocket) 신설

## 목표
koreanFoodDB.js 데이터를 활용한 "한식가이드" 포켓을 새로 만든다.
기존 RestaurantPocket(식당 회화)과는 별개의 독립 포켓이다.

## 데이터
- 파일: `src/data/koreanFoodDB.js`
- export: `koreanFoodDB` (배열, 126개 메뉴), `foodCategories` (16개 카테고리)
- 각 메뉴 항목 구조: id, ko, zh, en, category, spicy(0-5), price(₩범위), description(ko/zh/en), recipe(재료+조리법), funFact, orderTip, image 등

## 구현 사항

### 1. KoreanFoodPocket.jsx 생성
위치: `src/components/pockets/KoreanFoodPocket.jsx`

**UI 구조 (Grab/Toss 스타일):**
- 상단: 검색바 (한/중/영 통합검색)
- 카테고리 가로 스크롤 칩 (foodCategories 16개, '전체' 기본 선택)
- 메뉴 카드 그리드 (2열)
  - 카드: 음식 이름(ko/zh), 카테고리 배지, 맵기 🌶️ 표시(spicy level), 가격대
  - 카드 배경은 깔끔한 흰색, 그림자 최소
- 카드 터치 → 상세 모달/슬라이드업:
  - 음식 이름 (ko + zh + en)
  - 한줄 소개 (description)
  - 맵기 레벨 시각화 (🌶️ 아이콘 개수)
  - 가격대 표시
  - **주문 꿀팁** (orderTip) — 실제 식당에서 이 메뉴 주문 시 유용한 팁
  - **레시피** (recipe) — 접이식(collapsed), 터치하면 펼침
  - **재미있는 사실** (funFact) — SNS 공유하고 싶게 만드는 문화 팩트
  - 닫기 버튼

**디자인 원칙:**
- 29CM/Toss 스타일: Inter 폰트, 검정 단색, 흰 배경, 넉넉한 여백(24-32px)
- 이모지/아이콘 최소화 (맵기 표시 🌶️만 예외)
- 카드 간 간격 16px, 카드 내부 패딩 16px
- 카테고리 칩: 선택 시 검정 배경+흰 글씨, 미선택 시 회색 테두리

### 2. PocketContent.jsx에 등록
- `koreanfood` (또는 `kfood`) id로 등록
- 서비스 탭에서 접근 가능하도록

### 3. pockets.js에 데이터 추가
- 한식가이드 포켓 메타데이터 추가 (이름: 韩食百科 / 한식백과, 아이콘: UtensilsCrossed 또는 ChefHat)

## 하지 말 것
- koreanFoodDB.js 데이터 파일 자체를 수정하지 마
- 다른 기존 컴포넌트를 건드리지 마 (PocketContent.jsx, pockets.js 등록만)
- 이미지 URL이 없는 항목은 카테고리별 기본 색상 배경으로 처리
