## Phase 1: 직속직원 팀 리서치

각 C레벨 산하 7명이 **외부 시장을 조사**:

### 조사 방법
- **web_search + web_fetch 도구 적극 활용** — 실제 데이터 검색 및 사이트 직접 접근
- 샤오홍슈/도우인 한국여행 트렌드, 경쟁앱, 슈퍼앱 벤치마킹
- 중국인이 한국에서 겪는 실제 불편함 (커뮤니티, 블로그, 리뷰)

### ⛔ 데이터 원칙 (절대 규칙)
- **Mock/더미/가짜 데이터 절대 금지**
- **AI가 추정하거나 지어낸 수치 절대 금지** — "결제 실패율 37.2%" 같은 걸 검색 없이 만들어내지 마
- **모든 수치에 출처 URL 필수** — 출처 없으면 수치 사용 금지
- 데이터를 못 찾으면 솔직하게 "데이터 없음, 추가 조사 필요"로 표기
- **"~로 예상된다", "~로 추정된다" 금지** — 팩트만

### 안건 도출 시 필수 포함사항
1. **근거지표**: web_search로 찾은 실제 수치 + 출처 URL
2. **우리 서비스 현황**: 위 서비스 구조에서 관련 부분이 어디인지
3. **적용 방안**: 어떤 탭/컴포넌트/파일에 어떻게 넣을 건지 구체적으로
4. **적용 예시**: 실제 UI가 어떻게 보일지 텍스트/와이어프레임으로 묘사
5. **기대 효과**: 근거 기반으로만 (추정 금지, 없으면 "효과 측정 필요")

## Phase 3: 실행

### 즉시 반영 시 규칙
- **눈에 보이는 변화를 만들어라** — 린트/import 정리 같은 건 하지 마
- **Mock/더미/추정 데이터 절대 금지** — 가짜 데이터, placeholder, 하드코딩된 예시, AI가 지어낸 수치 넣지 마. 실제 검증된 데이터만. 데이터가 없으면 수집 방안을 제안만 하고 코드에 가짜 넣지 마. 리포트의 근거지표도 마찬가지 — 검색해서 찾은 실제 수치 + 출처 URL 없으면 수치 쓰지 마.
- 수정 후 `npx vite build` 빌드 확인
- 빌드 깨지는 수정 금지
- **배포 금지** (시월이 수동 배포)
- **telegram bot API 직접 호출 절대 금지**

### 신규 기능 제안 형식
```
### [기능명]
- **근거**: (시장 데이터, 수치)
- **현재 상태**: 우리 서비스에서 관련 부분이 어디에 있고 뭐가 부족한지
- **적용 방안**: 어떤 탭(home/service/course/korean)의 어떤 컴포넌트에, 어떤 형태로
- **적용 예시**: 유저가 보는 화면 묘사 (텍스트 와이어프레임)
- **구현 파일**: 수정/생성할 파일 목록
- **효과**: 예상 수치
- **난이도**: 상/중/하
```
