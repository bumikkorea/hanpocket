# HanPocket 마스터 브리핑 문서

> 원본: 범범뻠이 다른 AI에게 작성 의뢰한 전략/개발 마스터 문서 (2026-03-01)

## 핵심 포지셔닝
- 바이두/고덕지도 = "어떻게 가느냐" (교통)
- 한포켓 = "도착한 후 어떻게 하느냐" (생활 인프라)
- 카카오맵이 안 하는 이유: 한국인한테 필요 없어서

## Layer 프레임워크
- +layer: 급할 때 도와주는 것 (화장실, 약국, 택시, 응급, ATM, 와이파이)
- -layer: 나머지 전부 (관광, 맛집, 숙박, 쇼핑, 축제)
- +layer 먼저 → 신뢰 → -layer 채움

## 데이터 3층 구조 (해자/moat)
- 1층: 공공데이터 (진입장벽 없음)
- 2층: 사용자 데이터 ("여기서 알리페이 됐어요")
- 3층: 커뮤니티 데이터 ("직원이 중국어 통해요")

## 🔥🔥🔥 MVP 필수 (6개)
1. 제로페이 가맹점 = 알리페이 결제 가능 (+4)
2. 관광안내소 중국어안내가능여부 (-4)
3. 외국인환자 유치기관 (+5)
4. TourAPI 4.0 26만건 다국어 (-1~-5)
5. 비짓서울 API 3만건 7개국어 (-1~-3)
6. 인천공항 실시간 (진입점)

## 🔥🔥 유용 (8개)
7. 금융결제원 FIN MAP ATM (+4)
8. 환율 API (상시)
9. 기상청+에어코리아 (상시)
10. 전국공중화장실 (+3) ✅이미 적용
11. 전국약국정보 (+4)
12. 무슬림 친화식당 (-3)
13. 세계음식점 데이터 (-3)
14. 공공 와이파이 (+3~+4)

## 🔥 후순위 (10개)

## 핵심 원칙
1. +layer 먼저
2. 번역 비용 0원 전략 (TourAPI/비짓서울 이미 번역됨)
3. 80점으로 빨리 론칭
4. "알리페이 되는 곳", "외국인 가능 병원", "중국어 통하는 곳" = 카카오맵에 없는 화면
5. 공공데이터는 1층. 해자는 2-3층. 하지만 1층 없으면 2-3층도 못 쌓음.

## 좌표계 주의
- 대부분: WGS84 → 바로 사용
- 일부 (전국일반음식점): Bessel → WGS84 변환 (proj4js)
- 좌표 없는 데이터: 카카오 지오코딩 (일 100,000건 무료, 초과 0.5원/건)
