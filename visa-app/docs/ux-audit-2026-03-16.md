## Critical (즉시 수정)

### C-1. 만료된 팝업 5건이 여전히 데이터에 존재
`src/data/popupData.js`의 `POPUP_STORES`에 오늘(2026-03-16) 기준 이미 종료된 팝업 5건이 남아 있음.
사용자가 만료된 팝업 정보를 보고 직접 방문했다가 헛걸음할 수 있는 치명적 UX 오류.

|--------|--------|
| 조말론 잉글리시 페어 | 2026-03-15 |
| 나이키 에어맥스 팝업 | 2026-03-15 |
| 온유 TOUGH LOVE 팝업 | 2026-03-15 |
| 나이키스킴스 성수 팝업 | 2026-03-09 |
| 블랙핑크 DEADLINE 팝업 | 2026-03-09 |

**해결:** `popupData.js`에서 해당 5건 제거 또는 `isActiveOrUpcoming()` 필터를 모든 팝업 렌더링 경로에 강제 적용.

### C-2. 전체 팝업 이미지(25건) 100% Unsplash 플레이스홀더
`POPUP_STORES` 25건 전부 `images.unsplash.com` URL을 이미지로 사용 중.
실제 팝업 브랜드 이미지가 아닌 무관한 스톡 이미지가 노출되어 신뢰도가 크게 저하됨.
중국인 사용자는 위챗/샤오홍슈 수준의 실제 비주얼을 기대함.

**해결:** 각 팝업의 공식 SNS/네이버 블로그에서 실제 이미지 URL 수집하거나, Cloudflare R2에 업로드 후 대체.

## Warning (개선 권장)

### W-1. alert() 남용 — 모바일 UX 저해
`alert()` 네이티브 다이얼로그가 총 **19건** 발견. 모바일 PWA에서 `alert()`는 브라우저 UI를 덮고 iOS/Android에서 UX가 매우 거칠게 느껴짐.

주요 발생 파일:
- `MapTab.jsx` (7건) — 여행계획 한도 초과, 길찾기 에러
- `CourseTab.jsx` (4건) — GPS 오류, 저장 한도
- `JobsTab.jsx`, `HousingTab.jsx` — "준비 중" 안내
- `PlaceDetail.jsx`, `HalalGuide.jsx` 등

**해결:** 앱 내 토스트(toast) 또는 바텀시트 컴포넌트로 교체. 이미 `SOSTab.jsx`에 토스트 패턴이 구현되어 있으므로 재사용 가능.

---

### W-2. 사용자에게 직접 노출되는 "준비 중" / "Coming Soon" 텍스트 11건
핵심 기능들이 아직 미구현 상태로 사용자에게 그대로 노출됨. 중국 앱 수준의 완성도 기대에 맞지 않음.

주요 항목:
- `JobsTab.jsx:137` — "Coming Soon: 구직 등록" (구직자 등록 기능 미구현)
- `JobsTab.jsx:86` — 연락 버튼이 `alert("준비 중")`으로 연결
- `HousingTab.jsx:111` — 연락 버튼이 `alert("준비 중")`으로 연결
- `TravelTab.jsx:1246` — "Coming Soon" 하드코딩
- `travel/CityPage.jsx:143, 167` — "교통 정보는 준비 중" / "준비 중입니다"
- `pockets/MedicalPocket.jsx:38` — "위치 기반 약국 검색은 준비 중"
- `ARTranslateTab.jsx:13, 33` — AR 번역 전체가 "준비중" 상태

**해결:** 미구현 기능은 탭/버튼 자체를 숨기거나 비활성화(disabled + 흐림 처리). 사용자가 클릭했을 때 "준비 중" alert는 최악의 UX.
